# Scoring Model

The executable scoring path is:

```text
config/features.yaml + config/model.yaml
                  ↓
          factors/engine.py
                  ↓
       scoring/investment.py
                  ↓
Investment / Opportunity / Conviction
                  ↓
Deal Breakers / Decision / Thesis
```

The four factor scores are Business, Valuation, Financial and Timing.
`Confidence Score` is the model-coverage confidence produced by the active
factor engine; it is not the obsolete pre-scoring confidence calculation.

## Official live-scoring reference

Live watchlist, portfolio and `--ticker` scores use the latest versioned
`US_MARKET_ELIGIBLE` distribution generated by the broad-market screener. The
watchlist is no longer the percentile denominator. The artifact records
`reference_universe`, `reference_date`, `reference_count`,
`reference_version`, `model_version`, feature scopes and sorted distributions.

The current local reference was built from the 2026-07-13 snapshot: 6,959
successful observations were evaluated first and 2,429 eligible U.S. equities
formed the distribution. Failed and policy-ineligible instruments do not affect
the official percentiles.

Features marked `percentile_scope: sector` in `config/features.yaml` use the
company's sector distribution when at least five valid observations exist;
otherwise they fall back to the full eligible-market distribution. Timing and
Piotroski features remain market-relative.

If the artifact is missing or incompatible, Atlas preserves operability by
using the legacy current batch, but labels every row `CURRENT_BATCH`. Such
scores must not be compared with official-reference history. Historical
walk-forward remains cutoff-local and never consumes today's reference.

Refreshing the official reference can legitimately move scores because the
market distribution changed. Comparability therefore requires the four
reference metadata fields, not merely the model version. See ADR-012.

Configuration ownership:

- `config/features.yaml`: feature registry and feature-level weights;
- `config/model.yaml`: `factor_weights` — the factor-level weight vector;
- `config/deal_breakers.json`: governed risk-penalty rules and exemptions.

Changes to these files are financially material and require explicit tests and
documentation.
